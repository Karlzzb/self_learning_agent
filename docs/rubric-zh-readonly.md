# RUBRIC 中文速览(只读 · 非事实源)

本文件是 `RUBRIC.md` 的中文、无引用改写,**仅供阅读理解**。
它不是评分事实源,也不作后续开发参考——代码、judge 提示、忠实移植审计一律以英文的 `RUBRIC.md` 为准。
两者若有出入,以 `RUBRIC.md` 为准。

条目分三类:

- 确定性:由代码判定,通过/不通过,不交给 LLM。由课程创作子图的机器校验步把关(ADR-0006)。
- 判断:需人或 LLM 读完课程后打分,1–5 分(1=明显不达标,3=部分达标,5=完全示范)。
- 行为:不按单课打分,约束的是 agent 跨会话/工作区的行为,用于人评与相关节点的集成测试。

通过门槛:所有确定性条目必须 100% 通过,任一不过则整课不达标;每个判断条目必须≥3 分,且所有判断条目均值≥4.0。未过门槛的课退回起草步重写,直到最大重试次数;耗尽则按失败姿态处理(ADR-0009),不交付不达标的课。同一门槛也是人评的权威标准与 LLM judge 的校准目标。

## 第一部分 — 课程 rubric(L1–L17)

打分对象是一节产出的课(一个自足的 HTML 文件)。

### A. 契合 mission

L1 — 紧扣 mission(判断)。每节课都应直接扣住 mission,也就是学习者想学这个主题的真正理由。

L2 — 不漂出 mission(判断)。课程内容不偏移到学习者 mission 之外。

### B. 最近发展区(ZPD)

L3 — 单一紧凑范围(判断)。一节课是一个自足的 HTML 产物,只教一件扣住 mission、范围收得很紧的事。

L4 — 挑战刚刚好 / 落在工作记忆内(判断)。课程要短、能很快完成;学习者工作记忆很小,必须留在其中,让人始终觉得挑战刚刚好。

### C. 知识获取

L5 — 一个实实在在的收获(判断)。每节课给学习者一个可以往上累积的、实实在在的收获。

L6 — 每个论断都有引用(确定性:统计论断与引用,核对每条引用都能落到 `RESOURCES.md`)。课程应密布引用,每个论断都用外部资源背书。

L7 — 推荐一个最高质量的一手源(确定性:有/无)。每节课推荐一个供学习者读或看的一手源,应是你找到的最高质量、最可信的资源。

L14 — 只放技能所需的知识(判断)。课中的知识只放获取该技能所必需的部分。

### D. 技能与紧反馈

L8 — 基于反馈环的练习(针对技能)(判断)。技能通过互动课来教,每次练习都基于一个反馈环让学习者获得表现反馈;反馈环要尽可能紧,即时、且最好自动给出。

L9 — 测验选项不靠格式/长度泄露答案(确定性:标记为正确的选项不得是长度离群项)。测验各选项字数(尽量连字符数)应一致,不通过格式给出答案线索。执行说明(我们新增,语义优先):逐字要求(字数完全相等)只作为起草提示传给模型;确定性门只执行其目的——正确项不因长度被泄露——即仅当正确项相对每一个干扰项都显著更长或更短时才判失败。强行要求完全等长会扭曲选项措辞,语义一致与表达力优先;细微的等长打磨交给自审/人评层。

L15 — 先教知识,再练技能(判断)。先教知识,再让学习者通过互动反馈环练习技能。

### E. 持久性 — 必要难度

L16 — 用必要难度(检索练习/间隔/交错)(判断)。用必要难度设计课程以建立长期记忆:检索练习(从记忆中回忆)、间隔(把练习分散到不同时间)、交错(把不同但相关的主题混在一起练,仅限技能练习)。

### F. 一致性与美感

L10 — 复用共享组件,看起来像一门课(判断)。复用是默认而非例外;动手前先读 `./assets/`,在已有组件上搭建;每节课都链同一套件,让所有课看起来像一门连贯的课而非一堆散活。

L11 — 美观、Tufte 般清晰、值得回看(判断,可能需要看截图)。课程要美观,排版清晰易读,因为学习者日后会回看,向 Tufte 看齐。

L17 — 遵循 GLOSSARY 术语(可程序化比较处为确定性,否则为判断)。一旦建立词表,每节课都遵循它的术语;定义里也用词表自己的词,术语一旦入表就处处优先使用。

### G. 必备元素

L12 — 锚链到其他课/参考文档(确定性:有/无)。每节课用 HTML 锚点链到其他课与参考文档。

L13 — 提醒向 agent 追问(确定性:有/无)。每节课含一句提醒,鼓励学习者向 agent 追问;agent 是他的老师,能帮他弄懂任何不清楚的地方。

## 第二部分 — agent 行为标准(P1–P7)

不按单课打分;这些约束的是 agent 跨会话/工作区的行为。用于审查 agent 的举止,以及相关节点的集成测试。

P1 — 绝不轻信参数化知识,知识来自高可信资源(行为)。绝不轻信模型自身的参数化知识;知识应先从可信资源采集,并用 `RESOURCES.md` 记录。

P2 — RESOURCES.md 纪律(行为)。只收高可信源;每条都加注;按知识/智慧分组;明确标出空白;无情裁剪;记录社区偏好。

P3 — Mission:不清就访谈,变更时更新+记录+确认(行为)。若学习者对 mission 不清,或 `MISSION.md` 未填,你的第一件事是追问他为什么想学;mission 会随学习者成长而改变,这很正常,要更新 `MISSION.md` 并补一条学习记录来捕捉变化,且变更 mission 前先与学习者确认。

P4 — 智慧:先尝试作答,再引向高声望社区,尊重退出意愿(行为)。默认姿态是先尝试作答,但最终引向一个社区;应帮学习者找到可加入的高声望社区;若学习者表示不想加入社区,尊重其意愿。

P5 — 产出参考文档,全程遵循词表(行为)。创作课程的同时也要创作参考文档;参考文档应是课程的压缩精华,采用便于快速查阅的格式。

P6 — 学习记录纪律(证据级,不是流水账)(行为)。满足以下任一才写:展示了真正的理解、披露了既有知识、纠正了一处误解、mission 发生了转变;仅仅"讲过"的材料不算——覆盖不等于学会,要等到有证据。

P7 — 词表纪律(理解后才加;有主见;定义精简)(行为)。学习者真正理解某术语后才把它加入词表;要有主见;定义保持精简。
