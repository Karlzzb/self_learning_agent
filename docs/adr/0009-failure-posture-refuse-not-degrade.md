# 失败姿态:宁缺毋滥,状态不丢

当外部依赖失效时,agent 采取"教学诚实"优先的失败姿态:**宁可暂缓交付,也不交付未达标或未经资源核实的教学产物**,且任何失败都先保证 checkpointer 已存档、学习者可无损续学。

## 背景与理由

teach 在 Claude Code 里由宿主兜住搜索失败/模型超时;独立 agent 必须自定义面向学习者的失败姿态。具体:

- **搜索源够不着** → 坦白告知"暂时查不到可信资料、本节课先不做",**不**降级用模型脑补知识 —— 因为后者直接违反 SKILL.md "Never trust your parametric knowledge"。
- **Lesson 质量门(ADR-0006)重写到最大次数仍不达标** → 兜底为"不交付、请学习者稍后再来",而非输出未达标版本。
- **模型/网络硬故障** → 确定性要求:先确保状态已存档,下次无损续学。

## 影响

- 这是**刻意的"拒绝而非降级"**:未来读者若看到"查不到就停"的逻辑,不要将其当作缺陷改成降级输出 —— 这是为忠于 teach 的诚实原则而做的取舍。
- "最大重试次数 + 兜底"是健壮性硬性要求,不是可选项。
